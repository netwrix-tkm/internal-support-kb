## Problem Description
The customer requested assistance in generating a daily block report that would be sent via email. The report needed to include details such as who sent what, who it was sent to, the blocking policy, and the exact time of the action.

## Environment Details
- The issue was related to the Netwrix Endpoint Protector platform, specifically focusing on content-aware protection and predefined content (PII).

## Troubleshooting Steps
1. Reviewed the customer's request for clarity on the report requirements.
2. Checked the existing reporting capabilities within the Netwrix Endpoint Protector.
3. Explored the configuration settings for automated report generation and email notifications.
4. Verified if the necessary permissions and settings were in place for the report to include the requested details.

## Root Cause
The root cause was identified as a lack of configuration for automated reporting within the Netwrix Endpoint Protector system. The system did not have the necessary settings enabled to generate and send the requested block report via email.

## Solution
The issue was resolved by guiding the customer through the configuration process to set up automated daily block reports. This included:
- Enabling the report generation feature in the Netwrix Endpoint Protector.
- Configuring the report to include the required fields: sender, recipient, blocking policy, and action time.
- Setting up the email notification settings to ensure the report is sent out daily.

## Notes
- Ensure that the report generation feature is properly configured for future requests to avoid similar issues.
